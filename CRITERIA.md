# アセスメントAI アセスメントスコアリング基準

本アセスメントでは、受検者の回答を以下の3つの軸で評価し、各モジュールで10点満点でのスコアを算出します。

## 評価軸

1.  **課題認識 (Problem Recognition)**:
    *   与えられた状況やデータから、本質的な問題や潜在的なリスクをどれだけ正確に、深く理解しているか。
    *   表面的な事象だけでなく、その背景にある構造や因果関係を捉えられているか。

2.  **行動具体性 (Action Specificity)**:
    *   課題に対する解決策や対応策が、どれだけ具体的で実行可能であるか。
    *   単なる方針表明に留まらず、誰が、何を、いつ、どのように行うかまで具体的に記述されているか。
    *   予期せぬ事態への対応策や、関係者への具体的なコミュニケーション内容が含まれているか。

3.  **論理と共感 (Logic and Empathy)**:
    *   提案や判断の根拠が、どれだけ論理的で客観的か（データ、事実に基づいているか）。
    *   同時に、関係者の感情や立場、組織の状況を理解し、配慮したコミュニケーションやアプローチができているか。
    *   論理一辺倒ではなく、人間関係や協力体制構築の重要性を認識しているか。

## モジュール別スコアリング詳細 (各10点満点)

### 🧩 Module 1: 業務管理（インバスケット）

**テーマ**: リソースマネジメントと「持続可能な」成果

| 評価項目           | 基準（高得点）                                                                                                                              | 基準（低得点）                                                                                                                            |
| :----------------- | :------------------------------------------------------------------------------------------------------------------------------------------ | :---------------------------------------------------------------------------------------------------------------------------------------- |
| **課題認識**       | 複数のタスクの緊急度・重要度・影響範囲を正確に把握し、リソース（時間、人員）の制約を認識している。タスクの相互関係も考慮している。         | タスクの緊急度・重要度を誤解している、またはリソース制約を考慮せず、単純な順番付けに留まっている。                                        |
| **行動具体性**     | 優先順位付けの根拠を明確にし、最初に着手するタスクと、その具体的な進め方・メッセージ内容を詳細に記述している。リスク（例：タスクBの再燃）への対策も提示。 | 抽象的な「まず〇〇します」に留まり、具体的な行動やコミュニケーション内容、リスク対策が不足している。                                        |
| **論理と共感**     | 論理的な優先順位付けと、関係者（上司、同僚）への配慮（協力要請、情報共有）をバランス良く行い、対話を通じて解決を図ろうとしている。           | 責任感から全てを抱え込んだり、逆に他者に丸投げしたりする。または、一方的な指示に留まり、関係者との協調性が見られない。                  |
| **特記事項**       | - **Pattern A (抱え込み)**: 責任感は評価するが、持続可能性や効率性に欠けるため、スコアは低め (3-5点)。                                          |                                                                                                                                           |
|                    | - **Pattern B (縦割り)**: 自身の役割を全うする姿勢は評価するが、他者との連携や全体最適の視点が不足しているため、中程度 (5-7点)。               |                                                                                                                                           |
|                    | - **Pattern C (委譲)**: 全体最適を意識した行動は評価するが、委譲後のフォローやリスク管理まで含めた具体的な行動計画で高得点 (7-9点)。         |                                                                                                                                           |

### 🗣️ Module 2: ピア・レビュー（対人影響力）

**テーマ**: 相手を動かす「伝え方」の工夫

| 評価項目           | 基準（高得点）                                                                                                                              | 基準（低得点）                                                                                                                          |
| :----------------- | :------------------------------------------------------------------------------------------------------------------------------------------ | :-------------------------------------------------------------------------------------------------------------------------------------- |
| **課題認識**       | 企画書の具体的な問題点（ターゲット、KPI、コンセプト）を正確に指摘し、かつAさんの性格（プライド、不機嫌になりやすい）を考慮している。        | 企画書の問題点が曖昧、またはAさんの性格を考慮せず、単に問題点を指摘するに留まっている。                                                 |
| **行動具体性**     | Aさんのプライドを傷つけずに、具体的な改善点（媒体選定、KPI設定）を促すためのメッセージ例やアプローチを詳細に記述している。客観的なデータや事例を引用。 | 抽象的な「やんわり伝える」に留まり、具体的なメッセージ内容や、Aさんの感情に配慮したアプローチが不足している。                              |
| **論理と共感**     | 企画書の論理的な問題点を指摘しつつも、Aさんの感情に深く共感し、信頼関係を維持・向上させながらWin-Winの関係を目指している。                | 論理だけでAさんを追い詰めたり、逆に迎合して本質的な問題を看過したりする。対人関係への配慮が欠けている。                                 |
| **特記事項**       | - **Pattern A (論破)**: 論理は優れているが、Aさんの感情への配慮が不足しているため、スコアは低め (3-5点)。                                     |                                                                                                                                         |
|                    | - **Pattern B (迎合)**: Aさんの感情への配慮は優れているが、企画書の本質的な問題解決に踏み込めていないため、中程度 (5-7点)。                 |                                                                                                                                         |
|                    | - **高得点**：データや市場状況を引用しつつ、Aさんの経験や知見を尊重し、共同でより良い企画にするという姿勢で臨む場合 (7-9点)。               |                                                                                                                                         |

### 📊 Module 3: データ分析（課題解決）

**テーマ**: データドリブンな仮説思考

| 評価項目           | 基準（高得点）                                                                                                                              | 基準（低得点）                                                                                                                            |
| :----------------- | :------------------------------------------------------------------------------------------------------------------------------------------ | :---------------------------------------------------------------------------------------------------------------------------------------- |
| **課題認識**       | 会社方針（UI改修）がデータと矛盾することを明確に指摘し、真の課題がシステム負荷であることを認識している。他の仮説の可能性も考慮。          | 会社の表層的な方針に安易に同意したり、データの一部にのみ注目し、全体像を捉えられていない。                                                |
| **行動具体性**     | 事実（Fact 1, 2, 3）に基づき、上司に提案する具体的な対策（例：サーバー増強、ボトルネック特定、負荷分散）と、その提案に至るまでの論理的な説明を記述。 | 抽象的な「データを見直す」に留まり、具体的な対策案や、上司への説得プロセスが不足している。                                                 |
| **論理と共感**     | データ（ファクト）を論拠として論理的な提案を行う。会社方針を尊重しつつも、データに基づき適切に異議を唱え、建設的な議論を促す姿勢が見られる。 | データに盲目的に従い、上司の立場や感情を考慮せず一方的に正論を主張したり、逆にデータに根拠がないまま上司の方針に迎合したりする。        |
| **特記事項**       | - **Pattern A (追随)**: 上司の意向を尊重する姿勢は評価するが、データに基づくクリティカルな視点が欠けるため、スコアは低め (3-5点)。           |                                                                                                                                           |
|                    | - **Pattern B (対立)**: データに基づく指摘は評価するが、上司への配慮や説得の工夫が不足している場合、中程度 (5-7点)。                       |                                                                                                                                           |
|                    | - **高得点**：データと論理で真の課題を提示し、かつ上司の顔を立てながら建設的な解決策を提案し、組織全体の合意形成を図る場合 (7-9点)。     |                                                                                                                                           |

## 総合フィードバックにおけるスコアリングの活用

*   各モジュールのスコアを総合し、受検者の全体的な傾向や強み・弱みを導き出す。
*   スコアが低いモジュールに対しては、具体的な学習テーマや行動変容を促す。
*   スコアが高いモジュールに対しては、その強みをさらに伸ばすための示唆を与える。
*   ユーザーが希望した場合に、各モジュールのスコアと詳細な評価基準を提示できるようにする。
